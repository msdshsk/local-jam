# Claude 連携（MCP）

local-jam は起動中、**読み取り専用の MCP サーバ**を内蔵します。これにより Claude（Claude Code 等）が「いま開いているキャンバスの画面構成」を読み取り、設計の相談に乗れます。

> MCP は基本「プル型」です。アプリから Claude へ通知を送るのではなく、**Claude が必要なときにキャンバスを読む**形になります。

---

## エンドポイント

- URL: `http://127.0.0.1:4319/mcp`
- 方式: Streamable HTTP / JSON-RPC 2.0（自前実装の最小サーバ）
- スコープ: **localhost のみ・読み取り専用**
- 起動: アプリ（`npm run dev` もしくはビルド済み）の起動中だけ応答します。起動ログに `MCP server: http://127.0.0.1:4319/mcp` が出ます。

データ源は**オートセーブの最新スナップショット**です（変更後 ~800ms でほぼリアルタイム。起動直後は前回の復帰データ）。

---

## 公開ツール

| ツール | 内容 |
|---|---|
| `describe_layout` | 画面ごと・上から順に部品を並べた**テキスト要約**。まずこれを読めば全体像が分かる |
| `get_document` | 完全なドキュメント JSON（`nodes` / `edges` / `viewport` / `meta`） |

`describe_layout` の出力例：

```
# ボード「autosave」  画面数:2 / 部品数:7

## 画面「ログイン」 (位置 0,0 / 600×500 背景:#ffffff)
  - 見出し「ログイン」
  - 入力欄[email]「メールアドレス」
  - 入力欄[password]「パスワード」
  - ボタン「ログイン」

## 画面「商品一覧」 (位置 800,0 / 900×600)
  - ヘッダ ロゴ:Shop / nav:[商品・注文・設定] / 右:ログアウト
  - テーブル 列[商品名・価格・在庫] 5行
  - ページネーション 5ページ
```

生の React-Flow JSON より、こちらの要約の方が LLM は画面構成を理解しやすくなっています。

## 接続手順（Claude Code）

1. アプリを起動しておく（`npm run dev`）。
2. 一度だけ MCP を登録する：

   ```bash
   claude mcp add -t http local-jam http://127.0.0.1:4319/mcp
   ```

   確認：

   ```bash
   claude mcp list   # local-jam: http://127.0.0.1:4319/mcp (HTTP) - ✔ Connected
   ```

3. **新しい Claude Code インスタンス**を起動する（重要）。
   Claude Code は**起動中のインスタンスには MCP を後から追加できない**ため、登録後に新インスタンスを立てると有効になります。
4. Claude に「local-jam の `describe_layout` を読んで、この機能の画面案を出して」のように依頼する。

---

## 補足・制限

- **読み取り専用**です。Claude がキャンバスを直接編集することはできません（将来の拡張候補）。
- サーバはアプリの **main プロセス**に常駐します。ポートは現状 `4319` 固定です。
- ファイル保存・画像取り込み等の Electron 依存機能は MCP の対象外（あくまでキャンバス内容の読み取り）。
